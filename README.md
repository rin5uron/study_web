# 🚀 GitHubの使い方

## GitHubへの更新方針

- **重要**: GitHubへの更新（プッシュ）は明示的な指示がある場合のみ行います
- 通常の作業はローカル環境でのみ行い、コミットまでを完了させておきます
- GitHubへの反映が必要な場合は「GitHubに反映して」と明示的に依頼します
- この方針により、変更内容を確認してから最終的にリモートリポジトリに反映できます

## GitHubにファイルをプッシュする方法

### 基本的なGitHubへのプッシュ手順
１ ファイル画面からgit管理画面にとぶ（左から３つ目のマーク）
２ コミットメッセージに変更点等を記載
３ ➕マークを押す
５  3点リーダからプッシュ（ソース管理タブの並び！）


全てのデータを保存したい時
１ターミナルを開く
２git add .
3 enter で　U→A
4 コミットメッセージ　全てのファイルをアップロード
５ 3点リーダからプッシュ

1. 変更したファイルをステージングエリアに追加
   ```
   git add <ファイル名>
   ```
   または全ての変更ファイルを追加
   ```
   git add .
   ```

2. コミットする（変更を記録する）
   ```
   git commit -m "変更内容の説明"
   ```

3. リモートリポジトリにプッシュする
   ```
   git push origin <ブランチ名>
   ```
   メインブランチの場合は
   ```
   git push origin main
   ```

### GitHubワークフロー
1. リポジトリのクローン: `git clone <リポジトリURL>`
2. 変更・編集作業
3. 変更の確認: `git status`
4. 変更のステージング: `git add .`
5. 変更のコミット: `git commit -m "メッセージ"`
6. 変更のプッシュ: `git push`

これでGitHubにファイルをプッシュする方法をマスターしました！ 

# 🔍 技術用語・ショートカット辞典

## 辞典ファイルについて
`memo/tech_glossary.md` に覚えた技術用語やショートカットを辞書形式で記録しています。
以下のような特徴があります：

- **カテゴリ別の整理**
  - Git関連用語
  - Cursorショートカット
  - Macショートカット（追加）
  - HTML/CSS用語
  - ターミナルコマンド

- **表形式での管理**
  - 用語、意味、使用例、学んだ日時を表形式で整理
  - 時系列で学習過程が追跡可能
  - 視覚的に分かりやすく参照可能

- **検索機能**
  - 目次からカテゴリ別にジャンプ可能
  - ファイル内検索(Cmd+F/Ctrl+F)で素早く検索可能

- **更新方針**
  - 最初はリポジトリの用語のみを登録
  - 新しい用語やショートカットを学んだら、日時と共に随時追加
  - 目次も忘れずに更新

この辞典は技術学習の記録であると同時に、いつでも参照できる個人的なリファレンスとして活用します。
